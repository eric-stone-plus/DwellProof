# MAGI 协议 v1.0（原 QUINTE-LITE）

> Multi-Agent General Intelligence
> 2026-06-14 正式更名

---

## 设计哲学

MAGI 不是"三个AI给三个答案"，而是**三个独立视角交叉验证后给出一个可信结论**。

```
R1: hm(主审) + cc(审查者) + mc(宏观经济学家) — 独立分析
R2: 三方交叉互审 — 找分歧/找遗漏/找错误
R3: hm 终裁 — 综合三方论点，输出最终结论
```

## 角色定义

```
hm — 主审 (Arbitrator)
  职责: 综合判断、终裁、输出报告
  视角: 全局、平衡、实操导向

cc — 审查者 (Critic)
  职责: 独立审查、找漏洞、挑战假设
  视角: 对抗性、数据驱动、细节导向

mc — 宏观经济学家 (Macro)
  职责: 宏观分析、趋势判断、情景推演
  视角: 经济周期、政策传导、结构性分析
```

## 三轮流程

### R1 — 独立分析
三方各自读取相同数据，独立完成分析，不互相参考。
输出: 三份独立报告

### R2 — 交叉审
- hm 审 cc: 审查是否有统计谬误
- hm 审 mc: 审查宏观逻辑是否自洽
- cc 审 hm: 审查主审是否有遗漏
- mc 审 hm: 审查主审的宏观假设
输出: 三份交叉审报告

### R3 — 终裁
hm 综合三方论点，标注一致/分歧/独有发现，输出最终报告。
输出: 最终报告 + 仪表板 + 监测脚本

### R4 — 专家验证（可选）
邀请外部专家/数据源验证 R3 结论。

## 核心原则

1. **三方独立**: R1 不互相参考，避免共享盲区
2. **交叉验证**: R2 必须执行，不因 R1 一致跳过
3. **证据锚定**: 每个结论必须附数据/文件/行号
4. **分歧保留**: R3 必须标注分歧点，不得吞掉
5. **可追溯**: 所有中间结果保存到文件
