# Security and sensitive-data reporting

DwellProof handles a domain where reports may involve addresses, identity,
title, mortgage, and personal financial information. Never attach real case
data, credentials, or unredacted documents to a public GitHub issue.

For a vulnerability, use the repository's private vulnerability-reporting
channel when available. If it is unavailable, open a public issue containing
only a sanitized request for a private contact channel. Include no exploit
secret, personal record, exact property address, account value, or loan file.

For an accuracy issue, provide the smallest synthetic reproduction, the exact
formula or rule version, and public authoritative sources. A source correction
must not be converted into a directional property recommendation without the
complete case-specific evidence gate.

Only the current `main` branch is maintained, and it remains a controlled
prototype rather than an actionable investment-advice service.
