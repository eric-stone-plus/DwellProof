# Prototype audit - 2026-07-12

## Decision

The product direction is viable, but the former prototype outputs cannot be
wrapped and shipped. The useful asset is the accumulated domain research and
historical city-index dataset. The new product must be a property-level evidence
workbench with reproducible cash flows and explicit refusal conditions.

## Remediation status

The P0/P1 prototype defects listed below are retained as the original audit
record. The maintained paths now fail closed: heuristic forecast, timing, risk,
ranking, macro, and sell-guide generators refuse execution; legacy personal
state is quarantined; the monitor is descriptive only; and the new core binds
verified evidence to one case, property, and borrower and to a typed semantic
role. Readiness references are
revalidated as seven unique clear checks, while loan rows are recomputed against
the evidenced bank execution rate and repayment method. Missing borrower scope,
tiny unsupported nonzero rates, and conflicting reuse of an evidence ID across
scenario results now fail closed. Every legacy Markdown and HTML artifact also
has a first-screen non-actionable archive warning.

Final local verification on 2026-07-12 passed 77 core tests, 46 legacy safety
tests, the five-city official NBS replay, dependency audit, type checking,
production build, and desktop/mobile Playwright QA with zero console errors or
warnings. These repairs do not remove the release blockers at the end of this
document or turn the controlled prototype into investment advice.

## P0 findings

1. Core scripts referenced a removed absolute directory, so the project failed
   from its current location and the advertised test run passed only 4 of 7
   procedural checks.
2. The same monthly city index could trigger both "buy" and "sell". Neither path
   required property-level evidence.
3. "Investment ranking" was primarily a sort of handwritten rental yields. The
   underlying rent and price values were explicitly described as estimates.
4. The production sell rule and the sell backtest used opposite thresholds. The
   reported 70-85% hit rate therefore did not validate the displayed signal.
5. The lead/lag search initialized the best absolute correlation to 1, making a
   valid update impossible.
6. The only core price input was a city-level statistical index. It cannot price
   or recommend a specific home without comparable transactions and property
   attributes.
7. The update pipeline wrote and read different directories and printed success
   even when intermediate steps failed.

## P1 findings

- The valuation model used unsourced city prices and district/floor/orientation
  multipliers, silently defaulting unknown districts rather than refusing.
- The XGBoost experiment included contemporaneous target-derived features and
  therefore did not establish forward performance.
- A "successful" buy backtest meant the year-on-year index improved six months
  later; it did not measure a tradable return or transaction costs.
- The portfolio example multiplied a property value by a city index without an
  observation period or idempotency guard.
- The risk model used index levels as drawdown and kept macro risk constant.
- The tax calculator used the obsolete 90-square-metre split and a fixed 3%
  second-home rate. The current nationally confirmed split is 140 square metres
  for the policy effective 2024-12-01.
- The early-repayment example claimed five elapsed years while recalculating from
  the original principal and 360 remaining periods.
- `mega_analysis.py` could duplicate city/month rows when combining the main and
  per-city files.
- Static pages had no live data request. They displayed a June snapshot and used
  the 2008 PMI row (`53.0/60.2`) as if it were current; the CSV's 2026-05 row was
  `50.0/50.1`.

## Backtest interpretation

The reported average Shanghai lead-rule accuracy was 65.4%, but performance was
not uniform. A reproduced majority-class baseline beat the rule for Shenzhen
(`62.0%` baseline versus `54.2%` rule), and the improvement for Hangzhou was only
about 1.1 percentage points. This feature may remain city-specific market
context; it is not evidence of net investment return.

## Required release gate

Before any directional recommendation, DwellProof requires all of the following:

- recent, comparable property-level transactions with provenance;
- title holder and disposal authority checks;
- mortgage, seizure, objection, residence-right, and transaction-restriction checks;
- property nature, land use, lease, household registration, and school-place facts;
- versioned local tax rules and a written loan quote;
- a complete purchase/hold/exit cash-flow model;
- conservative/base/optimistic scenarios and a worst-case cash requirement;
- source timestamps, conflict handling, and a reproducible audit snapshot.

If any critical item is missing, the correct result is an evidence checklist,
not a softened buy/sell recommendation.
